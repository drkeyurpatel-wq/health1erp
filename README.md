# Health1ERP

## Architecture Overview
This project is designed to manage various operations in a hospital environment through a modular approach. Each module addresses core functionalities required for hospital management.

### Main Folder Structure:
- **patient_management/**: Handles all patient-related functionalities.
- **appointments/**: Manages appointment scheduling and tracking.
- **billing/**: Covers billing and invoicing processes.
- **inventory/**: Manages medical inventory and supplies.
- **staff/**: Keeps track of hospital staff and their roles.
- **core/**: Includes backend and frontend setups.

## Docker-Compose Setup
This project uses Docker for containerization to ensure smooth development and deployment.

### Local Development:
1. Clone the repository.
2. Run `docker-compose up` to start the services locally.

### Cloud Deployment:
Refer to the documentation for guidelines on deploying the application on cloud services.
